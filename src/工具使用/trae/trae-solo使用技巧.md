

TRAE 国内版的 Solo 模式是字节跳动推出的 AI 主导型智能编程模式，定位为 “The Responsive Coding Agent”，主打攻克项目迭代、架构重构等复杂开发任务，能让开发者从逐行编码转变为任务决策与进度把控角色，且当前对用户免费开放。

### Prompt定制：打造专属的指令模板
Cursor的AI效果核心依赖Prompt，**定制化Prompt模板**能避免重复输入相同指令，让AI直接生成符合你编码风格的代码。主要有两种定制方式：

1. **手动创建个人Prompt库**
   针对你常用的技术栈（如React、Python、Go），提前编写标准化的Prompt模板，存放在Cursor的笔记或本地文档中，使用时直接复制调用。**模板设计要点**：
   - 明确**技术栈+编码规范**：如`用React 18 Hooks+TypeScript写一个待办事项组件，遵循ESLint Airbnb规范，使用styled-components做样式，添加组件Props的类型校验`；
   - 指定**功能细节+边界条件**：如`用Go写一个Redis缓存工具类，支持Set/Get/Del操作，设置默认过期时间30分钟，处理Redis连接失败的重试逻辑（最多3次）`；
   - 定义**输出格式**：如`生成代码后，补充详细的函数注释和使用示例，用Markdown格式输出`。


TRAE国内版的Solo模式是字节跳动推出的AI主导型智能编程模式，定位为“The Responsive Coding Agent”，主打攻克项目迭代、架构重构等复杂开发任务，能让开发者从逐行编码转变为任务决策与进度把控角色，且当前对用户免费开放。下面为你详细介绍其核心功能及实用使用技巧：
### 核心功能介绍
1.  **SOLO Coder核心智能体**：这是该模式的核心，采用“主Agent - 子Agent”架构。勾选输入框旁的“Plan”选项，它会先输出开发计划而非直接编码，比如输入开发电商支付模块，会拆解出流程分析、接口设计等步骤。还能自动调度对应子智能体处理细分任务，也支持手动创建定制化子智能体，同时也支持添加新MCP集成。
![alt text](assets/trae-solo使用技巧/image.png)  

1.  **多任务并行开发**：TRAE国内版Solo模式支持多任务并行开发，可同时处理多个功能模块，每个功能模块可独立进行开发，互不影响。
2.  **代码变更查看（DiffView）**：点击对话流中操作步骤的文件卡片可单步追溯代码差异，完成功能模块后，点击对话流末尾的“查看变更”汇总卡片能整体审阅文件增改内容，还支持查看至多15个会话的代码变更记录，方便版本回溯。
3.  **上下文管理**：系统会智能压缩对话中的冗余信息，也可手动点击“压缩”按钮剔除无用内容。压缩后会清晰展示保留和舍弃的内容，避免AI在长对话中失焦。
4.  **三栏式清晰布局**：左侧是多任务窗口，用于创建和切换任务；中间是对话流区域，展示交互记录并支持摘要生成、进度标记；右侧是工具面板，集成代码变更查看、文件树等辅助工具。

### 实用使用技巧
1.  **复杂任务先开Plan模式**：处理重构代码、开发新功能等复杂任务时，务必勾选“Plan”选项。让AI先输出开发计划，若方案有漏洞可直接对话修改，确认后再执行，能大幅减少返工，比如修复复杂Bug前，可通过Plan明确问题原因和修复步骤。
2.  **灵活调度子智能体提效**：常规任务可依赖主Agent自动调度子智能体；有定制需求时，可输入如“数据库优化子智能体”的描述生成专属智能体，后续在提示词中指定调用，像处理包含图表的项目时，可创建“图表生成子智能体”专门负责相关工作。
3.  **合理拆分多任务并行**：面对大型项目，按模块拆分成多个小任务，比如将AI小说系统拆分为“爆款导语生成”“正文编写”等任务。这样既能提升开发效率，也便于单独管理每个模块的进度和质量。
4.  **定期审查代码变更**：每完成一个功能模块，及时通过DiffView查看代码变更。一方面能避免AI生成不符合架构规范的代码，另一方面若后续出现问题，可快速定位对应修改步骤，降低排查难度。
5.  **及时做上下文压缩**：多轮对话后或切换新功能模块前，手动触发压缩功能。剔除已解决Bug的讨论等冗余信息，既能让AI聚焦当前核心任务，还能节省Token消耗。
6.  **善用任务隔离与回溯**：遇到临时技术咨询或紧急Bug时，新建独立任务处理，避免干扰当前开发流程。同时对重要版本及时导出备份，防止超出15个会话后无法回溯变更记录的情况。
7.  **手动创建专属子智能体**：针对长期重复的专项任务，手动创建精准功能的子智能体。例如经常处理Vue组件开发，可创建“Vue组件开发子智能体”，后续相关任务指定调用该智能体，能提升开发适配度。


# TRAE Solo模式快速上手清单
## 一、核心功能速查（含入口+关键用法）
1. **SOLO Coder核心智能体**：输入框旁勾选“Plan”开启规划模式，手动创建子智能体直接输入功能描述，支持添加新MCP集成。
2. **多任务并行开发**：左侧多任务窗口点击“新建”拆分任务，切换任务直接点击对应窗口即可，互不干扰。
3. **代码变更查看（DiffView）**：单步差异点对话流中文件卡片，整体变更点对话流末尾“查看变更”，历史记录直接在工具面板查询。
4. **上下文管理**：手动压缩点对话区域“压缩”按钮，系统会自动展示保留/舍弃内容，无需手动筛选。
5. **三栏布局操作**：左栏管理任务、中栏交互记录（支持生成摘要/标记进度）、右栏工具面板（文件树+变更查看一体化）。

## 二、必用高效技巧（上手即提效）
1. 复杂任务先开“Plan”，确认开发步骤后再执行，减少返工。
2. 提示词明确技术栈+编码规范，比如“Vue3+TypeScript，遵循ESLint”。
3. 按模块拆分多任务，大型项目优先拆分为独立功能单元。
4. 完成模块后必查DiffView，规避架构不符问题。
5. 多轮对话/切换任务前，手动压缩上下文节省Token。
6. 临时需求/紧急Bug新建独立任务，避免干扰主流程。
7. 重复专项任务创建专属子智能体，搭配MCP集成使用。
8. 用对话标记管理开发进度：在对话流里用明确的标记划分阶段，比如 “【需求确认】”“【开发中】”“【测试通过】”。后续想回顾进度时，直接搜索标记就能快速定位关键节点，不用翻完整段对话。
9. 利用摘要功能快速同步进度：多轮对话后，点击对话流的 “生成摘要”，AI 会提炼核心进展、待办事项和已完成工作。不管是自己回顾，还是同步给团队成员，都不用逐字读对话记录。
10. 跟 AI 沟通时，把技术栈、编码规范说清楚，比如 “用 React 18 开发，遵循 ESLint 规范，组件命名采用 PascalCase”。这样 AI 生成的代码不用二次修改，直接对接项目现有框架，省掉格式调整的时间。

## 三、避坑指南（新手必看）
1. 不盲目接受AI代码，尤其是涉及权限、支付的核心模块，必须人工复核。
2. 重要版本及时导出备份，避免超出15个会话无法回溯变更。
3. 避免提示词模糊，比如不说“做个登录功能”，而说“做手机号+验证码登录，含表单校验”。
4. 不依赖AI调试全部问题，复杂逻辑需结合自身经验排查。
5. 子智能体无需创建过多，聚焦高频重复任务即可，避免管理冗余。

## 四、快速入门步骤（3分钟上手），最关键的是需求先行，代码后动，不然反复更改会有大量的历史残留
1. 启动后从欢迎界面点击“新建项目”，选择对应技术栈模板。
2. 输入首个开发需求，勾选“Plan”让AI生成执行步骤。
3. 确认计划后，让AI自动调度子智能体执行编码。
4. 完成后通过DiffView审阅代码，有问题直接在对话流修改。
5. 手动压缩当前上下文，新建下一个功能模块任务。
